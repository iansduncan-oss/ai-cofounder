    **1. Create `apps/agent-server/src/services/autonomous-executor.ts`:**

    Export `WorkLogAction` interface with fields: `type` (union of "task_progress" | "git_branch" | "git_add" | "git_commit" | "git_push" | "create_pr" | "error"), `timestamp: number`, and optional `taskId`, `taskTitle`, `agent`, `status`, `message`, `result: Record<string, unknown>`, `output: string`.

    Export `buildConventionalCommit(opts)` pure function. Accepts `{ type: "feat"|"fix"|"chore"|"refactor"|"test"|"docs", scope?: string, description: string, goalId: string, taskId?: string }`. Returns a conventional commit string: `"type(scope): description [goal:XXXXXXXX]"` or `"type(scope): description [goal:XXXXXXXX task:YYYYYYYY]"`. Enforces 72-char max subject line — truncate description with `...` if needed. Use first 8 chars of goalId/taskId for refs. Follow the exact pattern from 10-RESEARCH.md Pattern 3.

    Export `AutonomousExecutorService` class with constructor accepting `(dispatcher: TaskDispatcher, workspaceService: WorkspaceService | undefined, db: Db, registry: LlmRegistry)`.

    Implement `async executeGoal(opts)` method:
    - opts: `{ goalId: string, userId?: string, workSessionId: string, repoDir?: string, createPr?: boolean, onProgress?: TaskProgressCallback }`
    - Returns `{ progress: DispatcherProgress, actions: WorkLogAction[] }`
    - Flow:
      1. If workspaceService + repoDir: create feature branch `autonomous/${goalId.slice(0,8)}`, push git_branch action
      2. Call `this.dispatcher.runGoal(goalId, userId, progressCallback)` — record each progress event as a task_progress action (truncate output to 500 chars)
      3. If workspace + repoDir + progress.status === "completed": call `commitAndOptionalPr()`
      4. If getCostByGoal available: query cost and attach as costSummary to return value
    - `commitAndOptionalPr()` private method:
      1. Get goal title via `getGoal(db, goalId)`
      2. Run `gitStatus()` — if clean, push "Nothing to commit" action and return
      3. `gitAdd(repoDir, ["."])`, push git_add action
      4. Build commit message via `buildConventionalCommit({ type: "feat", description: goal.title, goalId })`
      5. `gitCommit(repoDir, commitMsg)`, push git_commit action
      6. If createPr: `gitPush(repoDir, "origin", branchName)`, push git_push action
      7. If createPr: call `generatePrDescription()` then `executeCreatePr()`, push create_pr action

    Implement `async generatePrDescription(goalTitle, progress)` method:
    - Use `this.registry.complete("conversation", ...)` with a system prompt "You write concise GitHub PR descriptions. Keep under 300 words."
    - User message: "Write a PR description for: \"{goalTitle}\"\n\nCompleted tasks:\n{taskSummaries}"
    - Extract text content from response, append "\n\n---\n*Autonomously generated by AI Cofounder*"
    - Follow exact pattern from 10-RESEARCH.md Pattern 2

    Use `GITHUB_DEFAULT_BRANCH` env var (via `optionalEnv("GITHUB_DEFAULT_BRANCH", "main")`) for PR base branch. Use `optionalEnv("GITHUB_REPO_OWNER", "")` and `optionalEnv("GITHUB_REPO_NAME", "")` for PR owner/repo — skip PR creation if either is empty and log a warning. Remember: `optionalEnv` requires 2 args (CLAUDE.md).
