<objective>
Close the FIN-01 verification gap: "Every LLM call has an accurate dollar cost persisted within the request lifecycle."

Currently only 3 of 15+ `registry.complete()` call sites record LLM usage. Instead of patching each site individually (fragile, error-prone for future additions), add an `onCompletion` callback hook to `LlmRegistry` that fires after every successful `complete()` call. Wire it once in `server.ts` to call `recordLlmUsage()`. Then remove the 3 existing manual recording sites to prevent double-counting.

Purpose: Ensure 100% cost tracking coverage for all LLM calls — orchestrator, specialists, briefing, journal, suggestions, reflection, summarizer, subagent, autonomous-executor, decision-extractor, deploy-health, memory-consolidation — with zero per-site maintenance burden.
Output: Modified registry with hook, server wiring, removed manual recording, tests, updated REQUIREMENTS.md.
</objective>
