    **6. Auth plugin (apps/agent-server/src/plugins/auth.ts):**
    Create a new Fastify plugin wrapped with `fp()` (global scope needed so `request.jwtVerify` is available everywhere):
    - Register `@fastify/cookie` with `secret: requireEnv('COOKIE_SECRET')`
    - Register `@fastify/jwt` with `secret: requireEnv('JWT_SECRET')` and `sign: { expiresIn: '15m' }`
    - Add `onReady` hook for admin seed:
      - Read `ADMIN_EMAIL` and `ADMIN_PASSWORD` via `optionalEnv()` with empty string defaults
      - If both are set AND `countAdminUsers(app.db) === 0`, hash password with `bcrypt.hash(password, 12)` and call `createAdminUser()`
      - Log the seed action with the admin email
    - Use `requireEnv` from `@ai-cofounder/shared` for JWT_SECRET and COOKIE_SECRET (server MUST fail to start if missing)
    - Augment Fastify types: add `jwtVerify()` on FastifyRequest and `jwtSign()` on FastifyReply (these come from @fastify/jwt but may need explicit declaration)

    **7. Auth routes (apps/agent-server/src/routes/auth.ts):**
    Create a Fastify plugin (no fp() wrapper needed — these are regular routes):

    **POST /login:**
    - Parse `{ email, password }` from request body
    - Call `findAdminByEmail(app.db, email)`
    - If not found or `bcrypt.compare(password, admin.passwordHash)` fails, return 401 `{ error: 'Invalid credentials' }`
    - Sign access token: `reply.jwtSign({ sub: admin.id, email: admin.email })` (15min expiry from plugin config)
    - Sign refresh token: `app.jwt.sign({ sub: admin.id, type: 'refresh' }, { expiresIn: '7d' })`
    - Set refresh token cookie: `reply.setCookie('refreshToken', refreshToken, { httpOnly: true, secure: process.env.NODE_ENV === 'production', sameSite: 'strict', path: '/api/auth/refresh', maxAge: 7 * 24 * 60 * 60 })`
    - Return `{ accessToken }`

    **POST /refresh:**
    - Read `request.cookies.refreshToken`
    - If missing, return 401 `{ error: 'No refresh token' }`
    - Verify with `app.jwt.verify(refreshToken)` — check `payload.type === 'refresh'`
    - Sign new access token with `reply.jwtSign({ sub: payload.sub })`
    - Return `{ accessToken }`
    - On any verify error, `reply.clearCookie('refreshToken')` and return 401

    **POST /logout:**
    - `reply.clearCookie('refreshToken', { path: '/api/auth/refresh' })`
    - Return `{ success: true }`
