  <name>Task 1: DB schema, migration, and repository functions for projects</name>
  <files>packages/db/src/schema.ts, packages/db/src/repositories.ts, packages/db/drizzle/0025_add_projects.sql, packages/test-utils/src/mocks/db.ts</files>
  <behavior>
    - createRegisteredProject inserts a project and returns the row with id, name, slug, workspacePath
    - listRegisteredProjects returns only active (isActive=true) projects ordered by name
    - getRegisteredProjectByName returns matching project or null
    - getRegisteredProjectById returns matching project or null
    - updateRegisteredProject updates fields and returns updated row
    - deleteRegisteredProject soft-deletes by setting isActive=false
    - createProjectDependency inserts a dependency and returns the row
    - listProjectDependencies returns all dependencies for a given project (both directions)
    - deleteProjectDependency removes the dependency row
  </behavior>
  <action>
    1. Add to `packages/db/src/schema.ts`:
       - `projectLanguageEnum` pgEnum with values: "typescript", "python", "javascript", "go", "other"
       - `registeredProjects` table: id (uuid pk), name (text unique not null), slug (text unique not null), repoUrl (text nullable), workspacePath (text not null), description (text nullable), language (projectLanguageEnum default "typescript"), defaultBranch (text default "main"), testCommand (text nullable), isActive (boolean default true), config (jsonb nullable), lastIngestedAt (timestamp tz nullable), createdAt (timestamp tz defaultNow), updatedAt (timestamp tz defaultNow)
       - `projectDependencies` table: id (uuid pk), sourceProjectId (uuid FK -> registeredProjects.id onDelete cascade), targetProjectId (uuid FK -> registeredProjects.id onDelete cascade), dependencyType (text not null), description (text nullable), createdAt (timestamp tz defaultNow)
       - Export both tables and the enum

    2. Add repository functions to `packages/db/src/repositories.ts` following existing patterns:
       - `createRegisteredProject(db, data)` — insert + returning
       - `listRegisteredProjects(db)` — select where isActive=true, order by name asc
       - `getRegisteredProjectByName(db, name)` — select where name, limit 1, return row or null
       - `getRegisteredProjectById(db, id)` — select where id, limit 1, return row or null
       - `updateRegisteredProject(db, id, data)` — update + returning, include updatedAt: new Date()
       - `deleteRegisteredProject(db, id)` — soft delete: update isActive=false + returning
       - `createProjectDependency(db, data)` — insert + returning
       - `listProjectDependencies(db, projectId)` — select where sourceProjectId=projectId OR targetProjectId=projectId
       - `deleteProjectDependency(db, id)` — hard delete + returning
       - Export all functions

    3. Create migration `packages/db/drizzle/0025_add_projects.sql`:
       - CREATE TYPE project_language (same values as enum)
       - CREATE TABLE registered_projects (all columns matching schema)
       - CREATE TABLE project_dependencies (all columns matching schema)
       - CREATE UNIQUE INDEX on registered_projects(slug)
       - CREATE UNIQUE INDEX on registered_projects(name)

    4. Update `packages/test-utils/src/mocks/db.ts` mockDbModule():
       - Add all 9 new repository function names as `vi.fn()` entries
       - This prevents downstream test files from breaking when they spread ...mockDbModule()

    5. Build the db package: `npm run build -w @ai-cofounder/db`

<task type="auto" tdd="true">
  <name>Task 2: ProjectRegistryService and Fastify plugin</name>
  <files>apps/agent-server/src/services/project-registry.ts, apps/agent-server/src/plugins/project-registry.ts, apps/agent-server/src/__tests__/project-registry.test.ts</files>
  <behavior>
    - registerProject creates WorkspaceService scoped to project.workspacePath and stores in internal map
    - registerProject rejects paths not under PROJECTS_BASE_DIR (throws error)
    - getWorkspace returns WorkspaceService for registered project ID, undefined for unknown
    - getActiveProject returns project data for given ID, null for unknown
    - listProjects returns all registered project metadata
    - loadFromDb loads all active projects from DB and registers their WorkspaceServices
    - validateProjectPath returns true for paths under allowed base dirs, false otherwise
    - Fastify plugin decorates app.projectRegistry and calls loadFromDb on register
  </behavior>
  <action>
    1. Create `apps/agent-server/src/services/project-registry.ts`:
       ```typescript
       export interface RegisteredProject {
         id: string;
         name: string;
         slug: string;
         workspacePath: string;
         repoUrl?: string;
         description?: string;
         language: string;
         defaultBranch: string;
         testCommand?: string;
         config?: Record<string, unknown>;
       }
       ```
       - `ProjectRegistryService` class:
         - Private `workspaces: Map<string, WorkspaceService>`
         - Private `projects: Map<string, RegisteredProject>`
         - Private `allowedBaseDirs: string[]` — initialized from `PROJECTS_BASE_DIR` env (comma-separated, defaults to `WORKSPACE_DIR`)
         - `validateProjectPath(workspacePath: string): boolean` — checks path starts with one of allowedBaseDirs. Use `path.resolve()` for normalization.
         - `async registerProject(project: RegisteredProject): Promise<void>` — validates path, creates WorkspaceService with workspacePath, calls init(), stores both
         - `getWorkspace(projectId: string): WorkspaceService | undefined`
         - `getActiveProject(projectId: string): RegisteredProject | undefined`
         - `listProjects(): RegisteredProject[]` — returns Array.from(projects.values())
         - `async loadFromDb(db: Db): Promise<void>` — calls `listRegisteredProjects(db)`, iterates results and calls registerProject for each. Catches per-project errors (log + skip) so one bad path doesn't block all.
       - Use `createLogger("project-registry")`
       - Use `optionalEnv("PROJECTS_BASE_DIR", optionalEnv("WORKSPACE_DIR", "/tmp/ai-cofounder-workspace"))` for base dir config

    2. Create `apps/agent-server/src/plugins/project-registry.ts`:
       - Fastify plugin via `fastify-plugin` (fp)
       - Declare module augmentation: `FastifyInstance.projectRegistry: ProjectRegistryService`
       - Instantiate `ProjectRegistryService`, call `registry.loadFromDb(app.db)` if app.db exists
       - Decorate app with `projectRegistry`

    3. Create `apps/agent-server/src/__tests__/project-registry.test.ts`:

       - Mock `node:fs/promises` mkdir (WorkspaceService.init calls it)
       - Test: registerProject stores workspace and project data

       - Test: getWorkspace returns undefined for unknown project
       - Test: loadFromDb registers all active projects from DB
       - Test: loadFromDb skips projects with invalid paths (logs warning, continues)
       - Test: listProjects returns all registered projects
       - Test: validateProjectPath returns true/false correctly
