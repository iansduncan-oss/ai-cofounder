Add two new `describe` blocks inside the existing `describe("WorkspaceService", ...)` block in `apps/agent-server/src/__tests__/workspace-service.test.ts`.

**describe("deleteFile"):**
1. "deletes a file within workspace":
   - `await workspace.writeFile("temp.txt", "content")`
   - `await workspace.deleteFile("temp.txt")`
   - `await expect(workspace.readFile("temp.txt")).rejects.toThrow()` (file gone)
2. "rejects path traversal on deleteFile":
   - `await expect(workspace.deleteFile("../../etc/passwd")).rejects.toThrow("Path traversal denied")`
3. "throws when deleting non-existent file":
   - `await expect(workspace.deleteFile("does-not-exist.txt")).rejects.toThrow()` (ENOENT)

**describe("deleteDirectory"):**
1. "deletes an empty directory":
   - `await fs.mkdir(path.join(testDir, "empty-dir"), { recursive: true })`
   - `await workspace.deleteDirectory("empty-dir")`
   - `await expect(workspace.listDirectory("empty-dir")).rejects.toThrow()` (directory gone)
2. "fails to delete non-empty directory without force":
   - `await workspace.writeFile("non-empty/file.txt", "content")`
   - `await expect(workspace.deleteDirectory("non-empty")).rejects.toThrow()` (ENOTEMPTY or similar)
3. "deletes non-empty directory with force=true":
   - `await workspace.writeFile("to-remove/nested/file.txt", "content")`
   - `await workspace.deleteDirectory("to-remove", true)`
   - `await expect(workspace.listDirectory("to-remove")).rejects.toThrow()` (directory gone)
4. "rejects path traversal on deleteDirectory":
   - `await expect(workspace.deleteDirectory("../../tmp")).rejects.toThrow("Path traversal denied")`

Place these new describe blocks AFTER the existing "listDirectory" describe block and BEFORE the "git core operations" describe block. The test file already has `fs`, `path`, `testDir`, and `workspace` available from the existing `beforeEach`.
