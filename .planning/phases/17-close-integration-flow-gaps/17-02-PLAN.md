<objective>
Create the Autonomous Sessions dashboard page and wire project switcher to workspace context.

Purpose: The autonomous execution engine (Phase 10) and scheduler (Phase 11) produce work sessions tracked in the DB and exposed via GET /api/autonomous/sessions, but no dashboard page exists to display them. The project switcher (Phase 16) stores an active project in localStorage but no page reads it — the workspace page should use it to set the browsing root.

Output: New Autonomous Sessions page with session list, strong typing, navigation wiring. Workspace page respects active project selection.

Note: This plan depends on Plan 01 because both modify `packages/api-client/src/types.ts`. Plan 01 adds `content_pipeline` to `JournalEntryType`; this plan adds the `WorkSession` interface. Sequential execution avoids file conflicts.
</objective>

     ```typescript
     const activeProjectId = useActiveProject();

       return project?.workspacePath ?? ".";
     }, [activeProjectId, projects]);
     ```
