Build the CI self-healing service that detects persistent CI failures and autonomously triggers fix sessions.

Output: CiSelfHealService with Redis-backed failure state tracking, monitoring integration, and autonomous fix session triggering. Comprehensive test coverage.

**1. Create CiSelfHealService (`apps/agent-server/src/services/ci-self-heal.ts`):**

- Import `Redis` from `ioredis`, `createLogger` from `@ai-cofounder/shared`
- Define `CiHealState` interface: `{ count: number; firstFailedAt: string; lastFailedAt?: string; lastWorkflowUrl?: string; healAttempted: boolean }`
- Export `CiSelfHealService` class:
  - Private constants: `HEAL_KEY_PREFIX = "ci-heal:"`, `FAILURE_THRESHOLD = 2`, `KEY_TTL_SECONDS = 7 * 24 * 3600` (7 days)
  - Constructor takes `redis: Redis` and `notificationService: NotificationService` (import type from `./notifications.js`)
  - Private `failureKey(repo: string, branch: string): string` returns `${HEAL_KEY_PREFIX}${repo}:${branch}`

  - `async recordFailure(repo: string, branch: string, workflowUrl: string): Promise<void>`:
    - Early return (skip) if `branch.startsWith("autonomous/") || branch.startsWith("dependabot/")` -- prevents infinite heal loops
    - Read existing state from Redis: `const raw = await this.redis.get(this.failureKey(repo, branch))`, parse or create default `CiHealState`
    - Increment `count`, update `lastFailedAt`, `lastWorkflowUrl`
    - Write back with `SET key JSON EX ttl`
    - If `count >= FAILURE_THRESHOLD && !state.healAttempted`:
      - Set `healAttempted = true`, write back again
      - Call `this.triggerHealSession(repo, branch, state)`

  - `async recordSuccess(repo: string, branch: string): Promise<void>`:
    - Delete the Redis key: `await this.redis.del(this.failureKey(repo, branch))`

  - `async getState(repo: string, branch: string): Promise<CiHealState | null>`:
    - Read and parse, return null if not found

  - Private `async triggerHealSession(repo: string, branch: string, state: CiHealState): Promise<void>`:
    - Send notification via `this.notificationService.sendBriefing()` with details about the failure (repo, branch, count, firstFailedAt, lastWorkflowUrl)
    - Dynamically import `enqueueAutonomousSession` from `@ai-cofounder/queue`
    - Call `enqueueAutonomousSession({ trigger: "ci-heal", prompt: "..." })` with a detailed prompt that instructs the agent to: fetch failing workflow jobs via GitHub API, identify root cause from step names, fix the code, commit on a new branch, create a PR, and use `request_approval` tool (yellow-tier) before merging
