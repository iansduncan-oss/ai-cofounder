**Step 1: Install ioredis.**
Run `npm install ioredis@5.9.3 -w @ai-cofounder/queue`. This pins to the same version BullMQ bundles internally, avoiding interface mismatches. Then run `npm run build -w @ai-cofounder/queue` to confirm the import resolves.

**Step 2: Create `packages/queue/src/pubsub.ts`.**
Follow the research pattern (02-RESEARCH.md Pattern 1) closely. The module exports:

- **Types:** `AgentProgressEvent` (task-level: goalId, goalTitle, taskId, taskTitle, agent, status, completedTasks, totalTasks, output?, timestamp), `AgentLifecycleEvent` (job-level: goalId, type, timestamp, error?), `AgentEvent` (union of both).
- **Constants:** `CHANNEL_PREFIX = "agent-events:goal:"`, `HISTORY_PREFIX = "agent-events:history:"`, `HISTORY_TTL_SECONDS = 3600`.
- **Helper functions:** `goalChannel(goalId)` returns `${CHANNEL_PREFIX}${goalId}`, `historyKey(goalId)` returns `${HISTORY_PREFIX}${goalId}`.
- **Class `RedisPubSub`:** Constructor takes `ConnectionOptions` (from BullMQ, same shape as `getRedisConnection()` returns). Creates one ioredis `Redis` instance (`this.publisher`). Methods:
  - `publish(goalId, event)` — calls `Promise.all([publisher.publish(channel, payload), publisher.rpush(historyKey, payload), publisher.expire(historyKey, TTL)])`.
  - `getHistory(goalId)` — calls `publisher.lrange(historyKey, 0, -1)`, parses each JSON string to `AgentEvent`.
  - `close()` — calls `publisher.quit()`.
- **Function `createSubscriber(connectionOptions)`** — creates a separate ioredis `Redis` instance for subscribing (separate from the publisher because Redis protocol requires dedicated connections for subscribe mode). Logs errors on the connection. Returns the Redis instance.

**Step 1: Update `apps/agent-server/src/worker.ts`.**
Import `RedisPubSub` from `@ai-cofounder/queue`. After `getRedisConnection(redisUrl)` call, instantiate `const redisPubSub = new RedisPubSub(getRedisConnection())`.

In the `agentTask` processor:

1. Before `dispatcher.runGoal()` — publish a lifecycle event:
   ```
   await redisPubSub.publish(goalId, { goalId, type: "job_started", timestamp: Date.now() })
   ```

2. Pass an `onProgress` callback as the third argument to `dispatcher.runGoal(goalId, userId, onProgress)`. The callback receives the dispatcher's progress event object (matches `TaskProgressCallback` type: goalId, goalTitle, taskId, taskTitle, agent, status, completedTasks, totalTasks, output?) and publishes it:
   ```
   async (event) => {
     await redisPubSub.publish(goalId, { ...event, timestamp: Date.now() });
   }
   ```

3. After successful `dispatcher.runGoal()` — publish completion lifecycle event:
   ```
   await redisPubSub.publish(goalId, { goalId, type: "job_completed", timestamp: Date.now() })
   ```

4. In the catch block (before `throw err`) — publish failure lifecycle event:
   ```
   await redisPubSub.publish(goalId, { goalId, type: "job_failed", error: err.message, timestamp: Date.now() })
   ```

5. In the shutdown handler, after `closeAllQueues()` — add `await redisPubSub.close()` to clean up the publisher Redis connection.
