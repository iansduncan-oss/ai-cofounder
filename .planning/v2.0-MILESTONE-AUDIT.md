## Integration Gaps

### INTEG-01: No Dashboard Consumer for Autonomous Execution API (TERM-01, TERM-05)

**From:** Phase 10 autonomous execution endpoints
**To:** Phase 16 dashboard
**Issue:** `GET /api/autonomous`, `GET /api/autonomous/sessions`, `POST /api/autonomous/:goalId/run` exist in server and ApiClient but have no dashboard page that calls them. Users cannot view work sessions or trigger autonomous runs from the UI.

### INTEG-02: content_pipeline Journal Type Missing from Dashboard (CONT-04, DASH-01)

**From:** Phase 15 pipeline journal entries
**To:** Phase 16 journal.tsx
**Issue:** `typeConfig` in `journal.tsx` has no `content_pipeline` key. Pipeline journal entries render with wrong icon ("Clock") and label ("Work Session" instead of "Content Pipeline").
**Fix:** Add `content_pipeline: { icon: Zap, color: "text-amber-500", label: "Content Pipeline" }` to `typeConfig`.

### INTEG-03: Project Switcher Not Wired to API Queries (PROJ-01, DASH-04)

**From:** Phase 16 `useActiveProject()` hook
**To:** Phase 16 API query hooks
**Issue:** `useActiveProject()` stores active project ID in localStorage but no dashboard API query passes it as a filter. The project switcher changes visual state but doesn't affect what data the dashboard shows.

### INTEG-04: Scheduler Bypasses Tier Enforcement (AUTO-01/02/03, SCHED-01)

**From:** Phase 11 scheduler.ts
**To:** Phase 9 autonomyTierService
**Issue:** `scheduler.ts` creates `Orchestrator` without passing `autonomyTierService` as 9th parameter. Scheduled cron-triggered sessions operate without tier enforcement — all tools are treated as green tier.
