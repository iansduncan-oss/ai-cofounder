## Critical Integration Gaps

### 1. SSE Stream + JWT Authentication Mismatch (Critical)

**Phases affected:** Phase 2 (SSE) + Phase 3 (JWT)
**Requirements affected:** AUTH-10, QUEUE-11

The dashboard's `useSSE` hook (`apps/dashboard/src/hooks/use-sse.ts` line 47) creates `new EventSource(fullUrl)` without auth credentials. The SSE endpoint (`GET /api/goals/:id/execute/stream`) is inside the `jwtGuardPlugin` scope. The browser `EventSource` API cannot send custom `Authorization` headers.

**Impact:** In production with `JWT_SECRET` set, all dashboard SSE streaming connections will fail with 401. Users will see "Connection lost" during goal execution.

**Fix options:**
1. Replace `useSSE` + `EventSource` with `apiClient.streamExecute()` (already exists, uses `fetch` with Bearer token)
2. Move SSE endpoint outside JWT guard and use cookie-based auth instead
3. Add `withCredentials: true` to EventSource and validate via refresh cookie on the SSE route

### 2. Bot Protected Route Access (Moderate)

**Phase affected:** Phase 3
**Requirement affected:** AUTH-09

Bot integrations calling protected routes (e.g., `POST /api/goals/:id/execute`) with only `API_SECRET` will get 401 in production. The `jwtGuardPlugin` only exempts routes outside its scope (channels, webhooks). Current bot commands work because they use `/api/channels/*` and `/api/webhooks/*` which are correctly outside the guard.

**Impact:** Low — current bot commands are unaffected. Only an issue if bots need to call execution or goal routes directly.
