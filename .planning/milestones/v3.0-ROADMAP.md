# Roadmap: v3.0 Production-Grade

**Milestone Goal:** Harden the AI Cofounder for daily reliable use — improve test infrastructure, deploy confidence, data scaling, and operational visibility.

**Requirements:** [v3.0-REQUIREMENTS.md](v3.0-REQUIREMENTS.md) — 24 requirements, 6 categories
**Predecessor:** v2.0 Autonomous Cofounder (43 requirements, 10 phases, shipped 2026-03-15)

---

## Phases

### Phase 18: Stabilization & Tech Debt
**Goal**: Address accumulated tech debt from v2.0 — type errors, constructor bloat, missing pagination, documentation gaps
**Depends on**: Nothing
**Requirements**: STAB-01, STAB-02, STAB-03, STAB-04, STAB-05, DASH-Q-01
**Status**: COMPLETE
**Success Criteria** (what must be TRUE):
  1. Orchestrator uses options object pattern (not positional args)
  2. `tsc --noEmit` passes clean on all agent-server files
  3. Deploy/pattern list endpoints return `{ data, total }` with real DB counts
  4. Phases 11 and 12 have SUMMARY documentation
  5. All tests pass (including pattern-analysis tests that were failing)
  6. Dashboard tests run in proper jsdom environment from monorepo root

### Phase 19: Integration Test Suite
**Goal**: Automated E2E flow testing that catches integration gaps unit tests miss — goal lifecycle, approval flows, API contracts
**Depends on**: Phase 18
**Requirements**: INTEG-01, INTEG-02, INTEG-03, INTEG-04
**Status**: COMPLETE
**Success Criteria** (what must be TRUE):
  1. Integration test harness boots agent-server with test DB and tears down cleanly
  2. Goal lifecycle test passes: create → plan → execute → verify → complete
  3. Approval flow test passes: yellow-tier tool → approval request → approve → resume
  4. Dashboard API contract tests validate all query hooks against real responses

### Phase 20: Deploy Pipeline Modernization
**Goal**: Reliable deployments with fast local feedback, deep health checks, and automated rollback on failure
**Depends on**: Nothing (independent)
**Requirements**: DEPLOY-01, DEPLOY-02, DEPLOY-03, DEPLOY-04
**Status**: COMPLETE (DEPLOY-04 dry-run deferred to v3.1)
**Success Criteria** (what must be TRUE):
  1. `npm run ci:local` runs the full GitHub Actions workflow locally via `act`
  2. `GET /health/deep` verifies DB, Redis, and at least one LLM provider connectivity
  3. Deploy script auto-rolls back if health check fails within 60s of deploy
  4. `npm run deploy:dry-run` builds images and validates config without deploying

### Phase 21: Operational Hardening
**Goal**: Production-grade data scaling and monitoring — server-side pagination for all large datasets, queue health visibility, validated budget enforcement
**Depends on**: Nothing (independent)
**Requirements**: OPS-01, OPS-02, OPS-03, OPS-04
**Status**: COMPLETE
**Success Criteria** (what must be TRUE):
  1. Journal, notification, and agent message list endpoints support `limit`/`offset` with DB-counted totals
  2. Queue health dashboard shows DLQ size, job completion rate, and stale job count
  3. Autonomous session respects budget hard-abort with real token counts (not estimates)
  4. Error reporting aggregates by type with frequency and last-seen timestamp

### Phase 22: Dashboard Quality & Documentation
**Goal**: Complete dashboard test coverage, accessibility improvements, and architecture documentation
**Depends on**: Phase 18
**Requirements**: DASH-Q-02, DASH-Q-03, DASH-Q-04, DOC-02
**Status**: COMPLETE (minor gaps deferred to v3.1)
**Success Criteria** (what must be TRUE):
  1. All dashboard hooks have unit tests using renderWithProviders
  2. Dashboard components use consistent test patterns
  3. Key pages pass basic accessibility checks (keyboard nav, ARIA)
  4. ADRs written for 3 key v2.0 architectural patterns
